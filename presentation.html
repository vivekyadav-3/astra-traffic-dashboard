🧠

ML Prediction Engine

Spatiotemporal ensemble model (ExtraTrees + LightGBM) forecasts demand in every geohash zone across the city for the next 30–45 minutes, updated every 15 minutes.

🗺️

Operator Dashboard

A browser-based command dashboard showing a live choropleth map of all 1,190 zones, timeline playback, zone inspection, mitigation simulation, and route delay estimation.

⚡

Zero Infrastructure

Predictions are pre-compiled into a lightweight JSON structure. The dashboard runs fully client-side — no server, no cloud cost, works on any tablet or PC in a control room.

🚦

Proactive, Not Reactive

Operators see congestion forming 30–45 minutes ahead and can simulate mitigation strategies (signal control, divert heavy vehicles, patrol dispatch) before the gridlock occurs.

Roadmap & Next Steps
From prototype to
city-scale deployment
🌐

ONNX Edge Inference

Compile ML models to ONNX and run predictions directly in the browser via WebAssembly — fully serverless production deployment.

🔗

Spatial Graph Modeling

Build geohash adjacency graph to propagate neighbor congestion as a feature — stepping stone to GNN-based forecasting.

🔄

Continuous Retraining

Automated daily/weekly model refresh with drift detection to keep predictions accurate as city traffic patterns evolve.

📡

Live Data Ingestion

Kafka streaming pipeline ingesting GPS probe data, camera counts, and weather API feeds for real-time prediction updates.
